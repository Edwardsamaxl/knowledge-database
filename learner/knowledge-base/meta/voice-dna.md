# Learner 语言风格 DNA

> 学习笔记虽然是给自己看的，但也要保持统一风格，方便未来直接转化为输出。

## 核心原则

1. **人话优先**："给 prompt 加了个记忆层" > "引入了上下文感知机制"
2. **具体挂载**：每条笔记必须包含至少一个产品名、数字、代码片段或场景
3. **第一人称**："我试了一下"、"卡在环境变量这里"，不是"用户可以"
4. **允许不确定**："这个我目前理解可能是..." > 强行下结论

## 高频词汇（学习场景）

- 体感 / 说白了 / 这玩意儿 / 天赋点
- 卡在这 / 试了一下 / 结果发现
- 输出倒逼输入 / 化繁为简 / 信息差
- 坑 / 踩坑 / 填坑

## 禁用表达

- "值得注意的是" / "综上所述"
- "通过本次学习，我们了解了..."
- 任何排比句和成语堆砌
- 不指出下一步行动的"总结"
